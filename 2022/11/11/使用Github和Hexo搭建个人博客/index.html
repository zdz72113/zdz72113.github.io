使用Github和Hexo搭建个人博客

发表于 2022-11-11 分类于 教程

相比于使用第三方博客网站不够自由，限制比较多。自己购买域名和服务器搭建博客购买成本高，需要定期维护。使用Github page平台托管我们的网站结合Hexo高效的本地管理功能，可以让笔者安心写作，无须担心文章审核或投入精力在网站维护上。

搭建

安装Git

Git是目前世界上最通用的分布式版本控制系统，可以有效的管理程序源代码及文档等。
Windows下请到官网下载并安装。

安装Node.js

Hexo是基于node.js编写的，所以需要安装一下node.js和里面的npm工具。
Windows下请到官网下载并安装。

在国内使用npm工具如果没有梯子的话，需要配置国内镜像进行加速。

安装Hexo

使用命令npm install -g hexo-cli安装Hexo，完成后通过hexo -v验证是否安装成功。

  1. 使用hexo init MyBlog初始化文件夹，把MyBlog替换为自己想定义的文件夹名称。
  2. 进入文件夹并安装组件。 目录下有：
    • node_modules: 依赖包
    • public：存放生成的页面
    • scaffolds：生成文章的一些模板
    • source：用来存放你的文章
    • themes：主题**
    • _config.yml: 博客的配置文件**
  3. 使用hexo g生成静态网页，然后输入hexo s打开本地服务器。然后浏览器打开http://localhost:4000/，就可以看到我们的博客啦，效果如下：

    按ctrl+c关闭本地服务器。

创建Github Page

打开https://github.com/，新建一个项目仓库New repository。
要创建一个和你用户名相同的仓库，后面加 .github.io，只有这样，将来要部署到GitHub page的时候，才会被识别，也就是http://xxxx.github.io，其中xxx就是你注册GitHub的用户名。例如我的：[http://zdz72113.github.io](http://zdz72113.github.io)

关联Hexo和GitHub

  1. 使用命令npm install hexo-deployer-git --save安装插件hexo-deployer-git
  2. 修改博客根目录下的_config.yml文件
  3. 发布文章到github 部署完成后，等待几分钟就可以在http://yourname.github.io 上看到博客了。

定制

hexo-theme-next是应用比较多的Hexo主题，主题的安装非常简单。

  1. 进入博客文件夹主目录执行git clone https://github.com/theme-next/hexo-theme-next themes/next下载主题，也可以手动下载后复制到themes文件夹下。
  2. 启用主题：打开_config.yml文件找到theme配置项并修改为next。
  3. next主题里面的具体配置请到\themes\next_config.yml文件里面做对应更改，比如定制外观，菜单，侧边框，开启打赏等，具体可参考http://theme-next.iissnan.com/getting-started.html

使用

  • 新建文章：hexo new post "article title"
  • 生成静态网页：hexo g
  • 本地预览：hexo s
  • 发布到Github：hexo d
  • 使用本地图片编辑带图文章需安装hexo-asset-image-for-hexo5
